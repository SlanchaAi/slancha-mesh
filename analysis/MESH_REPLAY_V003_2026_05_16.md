# Mesh Replay v0.0.3 — 2026-05-16

70-prompt slancha-test corpus driven through mesh service + live vLLM
on Spark GB10. Validates spec §12 day-7 end-to-end path (classifier →
mesh select → vLLM → response → trace) under real conditions, not mocks.

## Setup

| Component | What | Where |
|---|---|---|
| vLLM | `Qwen/Qwen3-Coder-30B-A3B-Instruct-FP8` (Marlin FP8 fallback for sm_121) | `127.0.0.1:8001` |
| Mesh service | `mesh.service:app` via `serve_with_catalog.py` | `127.0.0.1:8088` |
| Registry | `MeshRegistry(catalog=load_catalog())`, seeded with 1 heartbeat | in-process |
| Corpus | `corpus/v1/combined70.jsonl` from slancha-test | classified via `preclassify_corpus.py` |
| Replay harness | `mesh/scripts/mesh_replay.py` (paul-mac) | — |

## Corpus composition

```
math500   25 prompts → math
mtbench   25 prompts → general
mmlu      20 prompts → general
```

The pre-classifier (`mesh/scripts/preclassify_corpus.py`) maps id prefixes
to `{domain, difficulty, route_class}`. Difficulty derived from prompt
length (>800 hard, >300 med, else easy).

## Results

### Honest run — mesh has only Coder, corpus is math+general

`analysis/mesh_replay_v003_honest.jsonl`:

```
70 prompts replayed
 0 mesh-hit
70 cloud-fallback
```

Expected — none of the 70 prompts had `domain=code`, the only domain
covered by the in-flight specialist. Reason on every line:

```
no mesh route for math/easy/en (candidates=0 filtered_out=0); falling
through to cloud
```

This is the spec §6.6 contract working as intended: no coverage →
falls through fallback chain to cloud baseline.

### Forced-code run — classify all 70 as code/medium

Every prompt routed to `qwen3-coder-30b-a3b-fp8 @ http://127.0.0.1:8001`
with `queue_ms=0`. Validates the routing pipeline at full mesh hit rate
when coverage matches.

## Coverage gap, honestly

- One specialist in mesh today (`qwen3-coder-30b-a3b-fp8`); corpus is
  math/general; honest mesh-hit rate is 0/70 by design.
- The 70/70 forced-code result IS NOT a quality claim — Qwen3-Coder
  doesn't math-tune. It is purely a routing-pipeline validation.
- v0.0.4 expands the catalog with cached models that match corpus
  domains (Qwen3-Math-7B, Qwen3-8B general). Real mesh-hit on the
  unmodified corpus is gated on that catalog expansion.
- vs_cloud_baseline_cost field is a placeholder zero everywhere —
  Spark will fill it post-sprint when the streamlit panel lands.

## Reproducing

```bash
# Pre-flight: vLLM serving Qwen3-Coder-30B on :8001
bash mesh/scripts/bring-up-spark.sh qwen3-coder-30b-a3b-fp8 8001

# Boot mesh service on :8088 with catalog
python3 -c "
from mesh.service import create_mesh_app
from mesh.catalog import load_catalog
from mesh.registry import MeshRegistry
import uvicorn
uvicorn.run(create_mesh_app(registry=MeshRegistry(catalog=load_catalog())),
            host='127.0.0.1', port=8088)
" &

# Seed registry with one heartbeat for the live vLLM
# (see /tmp/mesh-live/seed_heartbeat.py for reference impl)

# Classify slancha-test corpus
python3 -m mesh.scripts.preclassify_corpus \
    --in  ~/Source/slancha-test/corpus/v1/combined70.jsonl \
    --out /tmp/prompts_classified.jsonl

# Drive replay
python3 -m mesh.scripts.mesh_replay \
    --corpus /tmp/prompts_classified.jsonl \
    --registry-url http://127.0.0.1:8088 \
    --output analysis/mesh_replay_v003_honest.jsonl
```
